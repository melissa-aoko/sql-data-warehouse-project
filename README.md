# 📊 Data Warehouse Project (SQL Server)

## 🧩 Overview
This project focuses on building a modern data warehouse using SQL Server to consolidate sales data from multiple sources. The goal is to enable efficient analytical reporting and support informed business decision-making.

---

## 🎯 Objective
Develop a structured, scalable data warehouse that integrates sales data and supports SQL-based analytics for business insights.

---

## ⚙️ Data Engineering (Data Warehouse Development)

### Specifications

**📥 Data Sources**
- Imported data from two source systems:
  - ERP system (CSV files)
  - CRM system (CSV files)

**🧹 Data Quality**
- Performed data cleansing to address:
  - Missing values  
  - Inconsistent formats  
  - Duplicate records  
- Ensured data reliability before analysis

**🔗 Integration**
- Combined ERP and CRM data into a unified, user-friendly data model  
- Designed schema optimized for analytical queries (e.g., star schema)

**📌 Scope**
- Focused on the most recent dataset only  
- No historization (no tracking of historical changes)

**📖 Documentation**
- Created clear documentation of the data model  
- Designed to support both:
  - Business stakeholders  
  - Data/analytics teams  

---

## 📈 BI & Analytics (Data Analysis)

### Objective
Develop SQL-based analytics to generate actionable insights into:

- Customer Behavior  
- Product Performance  
- Sales Trends  

These insights support stakeholders in tracking key business metrics and making data-driven decisions.
